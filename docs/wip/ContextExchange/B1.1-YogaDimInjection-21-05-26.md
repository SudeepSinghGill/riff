### This session: B1.1 ✅

**Commit `ba9869b`** — `fix(yoga): B1.1 inject measured cell dims before layout pass (L-7)`

**Root cause addressed:** Yoga's intrinsic measure function returns non-deterministic sub-pixel heights across commits (e.g., 335.3 → 334.7 → 335.3). This oscillated above the 0.5pt MVC threshold, triggering `applyMeasurements` → `finalizeHSection` → H-section wrapper height change → UIScrollView frame change mid-gesture → pan recognizer cancel → isDragging wedge state.

**Fix:**
- Override `layoutTree` in `CollectionViewContainerShadowNode` — the only hook that fires BEFORE `YGNodeCalculateLayout`.
- In `injectMeasuredDimensionsIfNeeded()`: for each `SizingState::Measured` cell, call `YGNodeStyleSetHeight(yogaChild, cachedH)` so Yoga returns the exact cached value. For `Placeholder` cells: `YGNodeStyleSetHeightAuto` to ensure intrinsic measurement runs.
- Accesses child yoga nodes via `yogaNode_.getChildren()` — protected member of `YogaLayoutableShadowNode`, accessible from derived class. Elements are `yoga::Node*` which is also a valid `YGNodeRef` (yoga::Node inherits from YGNode).
- Covers V-section cells (Height only, via layout engine's `contentDeterminedDimension`) and H-section grandchildren (Both axes, since CompositionalLayout returns `ContentDimension::Both`).

**Workarounds removed (no longer needed):**
1. `std::ceil` + 2pt hysteresis in `CompositionalLayout::finalizeHSection`
2. `std::ceil` + 2pt hysteresis in `CompositionalLayout::refreshHSectionWrapperHeight`
3. `_applyOrDeferScrollViewFrame` busy-guard + `_pendingScrollViewFrame` in `RNCollectionSubContainerView.mm`

## Key implementation note

`YGNodeStyleSetHeight` calls `markDirtyAndPropagate()` internally (only when value changes). If child is already dirty: no-op. If parent is already dirty: propagation stops at parent. So calling this in `layoutTree` before `ConcreteViewShadowNode::layoutTree` is safe — dirty flags are handled correctly by Yoga's layout algorithm.
