## What Was Built

Full horizontal grid layout. `columns=N` in horizontal mode means N items tile **top→bottom** (cross axis = Y) within a column-group, then advance **left→right** (primary axis = X). This is UIKit/CSS grid default for horizontal scroll.

Column-major fill order only — row-major deferred to flow layout (see PLAN.md R2).

---

## API

```typescript
layout={grid({
  horizontal: true,
  columns: 2,           // 2 items per column-group (tile top→bottom)
  rowHeight: 110,       // estimated item width (primary axis); Yoga measures final
  estimatedCrossAxisHeight: 120,
  columnSpacing: 8,     // gap between items within a column-group (along Y)
  rowSpacing: 4,        // gap between column-groups (along X)
  sectionBackground: true,
  separator: { height: 1.5 },
})}
```

Headers/footers are narrow vertical strips spanning full viewport height. Section backgrounds span the column-group extent. Separators: between-group = vertical lines; within-group = horizontal lines.

## Implementation

### C++ `GridLayout.cpp`

**`compute()`** — was hardcoded `_horizontal = false`. Fixed to read `params.horizontal` and `params.viewportHeight`.

**`computeSection()`** — full H-branching added:
- `const bool H = p.horizontal`
- `crossContent` = viewport cross-axis size minus section cross insets
- `itemCrossSize` = `(crossContent - totalColSpacing) / cols` — fixed, not Yoga-measured
- `itemPrimarySize` = `rowHeight` (fixed) or `estimatedCrossAxisHeight` (estimated, Yoga measures)
- Item frame for H: `{ rowStartPrimary, crossStart + col*(itemCrossSize+spacing), itemPrimary, itemCrossSize }`
- Header/footer for H: `{ primary, 0, headerH, viewportHeight }` — full cross-axis strip
- Between-group separator (vertical line): `{ rowEnd, crossStart+insetL, sepH, crossContent-insets }`
- Within-group separator (horizontal line): `{ rowStart, gap, rowMaxPrimary, sepH }`
- sizingState: `(H || !fixedPrimary) ? Placeholder : Measured`

**`computeSectionFromCache()`** — H-branching mirrors computeSection:
- Reads `frame.width` as primary-axis size for H (Yoga-measured widths)
- Cross-axis sizes (heights in H mode) always recomputed from column count — never read from cache

**`applyMeasurements()`** — H writes to `frame.width` (primary), V writes `frame.height`:
```cpp
const bool H = _horizontal;
if (H) updated.frame.width = d.newValue;
else   updated.frame.height = d.newValue;
```
Section start lookup uses `frame.x` for H, `frame.y` for V.

**`invalidateSectionsFrom()`** — same axis fix for section start position reads.

### TypeScript `protocol.ts` + `grid.ts`

- `GridLayoutDelegate`: added `horizontal?: boolean`, `estimatedCrossAxisHeight?: number`
- `GridLayoutEngine`: `readonly horizontal` field set from delegate
- `prepare()`: calls `nativeMod.layoutCache.setHorizontal(H)`, guards `itemHeights` to `!H`, passes `horizontal`, `viewportHeight`, `estimatedCrossAxisHeight` to C++
- `shouldInvalidate()`: checks height change for H, width change for V

Added `HorizontalGridDemo` component and `H-Grid` sub-tab:

- `columns: 2`, `rowHeight: 110`, sticky headers/footers, section backgrounds

## Fill Order Decision (captured in PLAN.md R2)

Column-major only. Row-major belongs in flow layout. Adding it later = one-line index formula swap. No prop needed for this grid engine.
